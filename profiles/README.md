# Profiles

Profiles are curated subsets of the rule library.

## Planned profiles

- `minimal.md` minimal safe baseline
- `default.md` recommended baseline for most repositories
- `strict.md` stronger control for sensitive repositories
- `solo-dev.md` practical setup for personal projects
- `automation-scripts.md` tuned for script-heavy repositories

## Profile intent

A profile should answer:

- which rules are enabled by default
- which rules are optional
- which rules are intentionally excluded
- what repository types the profile fits
