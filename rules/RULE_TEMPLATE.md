# Rule Template

Use this structure for each future rule file.

## Header

- Rule ID
- Title
- Domain
- Adoption level

## Logline

One short sentence that explains the rule.

## Scope

Examples:

- repo level
- pull request level
- coding session level
- release workflow level

## When to use

Trigger conditions for applying the rule.

## Required actions

What must be done when the rule applies.

## Exceptions

When the rule may be skipped.

## Affected artifacts

Which files, workflows, or outputs are usually touched.

## Automation

Which scripts, templates, or hooks support the rule.

## Notes

Short explanation in plain language for humans.
